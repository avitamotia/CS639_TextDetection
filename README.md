# Gemini-M4 dataset pipeline

Build a SemEval2024-M4 SubtaskA-shaped dataset where every machine sample is generated by `gemini-3.1-flash-lite`, plus an augmented test set for measuring detector robustness when frontier-model text is mixed in.

## Layout

```
m4_gemini_scripts/         # generation + experiment scripts
  m4_stats.py
  gemini_common.py
  generate_prompts.py
  generate_samples.py
  mix_with_humans.py
  append_gemini_to_test.py
  run_gemini_sweep.py

ood-llm-detect-main/data/gemini-M4/
  _intermediate/           # prompts, raw gemini outputs, resumable progress (gitignored)
  SubtaskA/                # final ratio-balanced train/dev/test
    subtaskA_{train,dev,test}_monolingual.jsonl
  full_test/               # original M4 test + all gemini test samples appended
    subtaskA_test_monolingual.jsonl
```

## Prerequisites

1. Original M4 data already present at `ood-llm-detect-main/data/SemEval2024-M4/SubtaskA/`.
2. Put your Gemini API key in `gemini-key.txt` at the repo root (one line, no quotes). It's gitignored.
3. Install deps:

   ```
   pip install -r requirements.txt
   ```

## Build the dataset

Run from the repo root:

```
python m4_gemini_scripts/m4_stats.py                    # one-time: counts the original splits
python m4_gemini_scripts/generate_prompts.py            # ~6500 disjoint prompts (train/dev = wikiHow, test = outfox-style)
python m4_gemini_scripts/generate_samples.py --split dev    # smallest split first; smoke test
python m4_gemini_scripts/generate_samples.py --split test
python m4_gemini_scripts/generate_samples.py --split train
python m4_gemini_scripts/mix_with_humans.py             # samples humans at original M4 ratio + writes SubtaskA/
python m4_gemini_scripts/append_gemini_to_test.py       # builds full_test/ (full original test + all gemini test)
```

Each step is idempotent / resumable. `generate_samples.py` writes one record per line as it succeeds and tracks completed prompt indices in a sidecar; re-run the same command to retry failed prompts.

Useful flags:

- `generate_samples.py --workers 8 --rpm 30 --limit 50` — concurrency, RPM cap, and a debug limit.
- All generators accept `--model <id>`. Run `python m4_gemini_scripts/gemini_common.py` to print every model id your API key has access to if `gemini-3.1-flash-lite` isn't accepted.

## Verify

```
python -c "import sys; sys.path.insert(0, 'ood-llm-detect-main'); from utils.M4_utils import load_M4; print({k: len(v) for k,v in load_M4('ood-llm-detect-main/data/gemini-M4/SubtaskA').items()})"
```

Expect roughly `monolingual_train ≈ 10.6k`, `monolingual_dev ≈ 1k`, `monolingual_test ≈ 1.9k`.

## Use the dataset

Two new training arguments in `ood-llm-detect-main/train_classifier_energy.py`:

| flag | default | purpose |
|------|---------|---------|
| `--n_gemini_train N` | `0` | mix N gemini-only samples into the M4 training set |
| `--gemini_train_path PATH` | `data/gemini-M4/_intermediate/gemini_train.jsonl` | source of those samples |
| `--eval_full_test_path PATH` | (off) | folder with a gemini-augmented test set; runs a second eval each epoch and logs metrics with `_full` suffix |

For the "does adding gemini training data help?" experiment, sweep N with the canonical settings:

```
python m4_gemini_scripts/run_gemini_sweep.py --epochs 10
# or pick a smaller set: --values 0 100 1000
# or just look at the commands: --dry_run
```

Each sweep run produces its own folder under `ood-llm-detect-main/runs/` with TensorBoard scalars (`val_auc`, `val_auc_full`, ...) and a `test_results_M4_energy.json` containing both canonical and augmented eval metrics for the best epoch.
